Πώς μπορώ να χρησιμοποιήσω συγκεκριμένο μήκος της γραμματοσειράς για όλο το κείμενο;

Με την χρήση αυτών των ιδιοτήτων:
font.base=$(font.monospace)
font.small=$(font.monospace)
font.comment=$(font.monospace)
font.text=$(font.monospace)
font.text.comment=$(font.monospace)
font.embedded.base=$(font.monospace)
font.embedded.comment=$(font.monospace)
font.vbs=$(font.monospace)

Τί απέγινε το use.monospaced;

Η use.monospaced ιδιότητα αφαιρέθηκε αφού οι άνθρωποι το χρησιμοποιούσαν για να έχουν γραμματοσειρά σταθερού μήκους και μετά απαιτούσαν υποστήριξη για να ξεπεράσουν τους περιορισμούς της ιδιότητας αυτής. Ο σωστός τρόπος χρήσης γραμματοσειράς σταθερού μήκους είναι εδώ.

Γιατί η ρύθμιση line.numbers δεν λειτουργεί;

Το line.numbers αντικαταστάθηκε με δύο ιδιότητες: το line.margin.visible και το line.margin.width τα οποία επεξηγήθηκαν νωρίτερα στην κύρια τεκμηρείωση του SciTE.

Πώς αλλάζω το φόντο στο SciTE για να βάλω μαύρο;

Πρέπει να αλλαχτούν οι ρυθμίσεις στυλ. Η κύρια αλλαγή είναι στο αρχείο γενικών ρυθμίσεων, στο style και caret χρώμα αλλά ίσως χρειαστούν αλλαγές και άλλων ρυθμίσεων για να λειτουργήσει σωστά:
style.*.32=$(font.base),back:#000000,fore:#ffffff
caret.fore=#FFFFFF

Πώς αλλάζω τα χρώματα στο παράθυρο αποτελεσμάτων(εξόδου);

Το παράθυρο αποτελεσμάτων(εξόδου) συχνά εμφανίζει μηνύματα λαθών και προειδοποιήσεις και το στυλ του καθορίζεται από το "errorlist" του lexer. Το προκαθορισμένο errorlist στυλ βρίσκεται στο στο αρχείο others.properties. Για να αλλαχτεί το φόντο σε μαύρο και το προκαθορισμένο χρώμα κειμένου σε λευκό πρέπει να οριστούν τα ακόλουθα:
style.errorlist.32=$(font.small),back:#000000
style.errorlist.0=fore:#FFFFFF

Πώς προσαρμόζω την οριζόντια μπάρα κύλισης στο μήκος του κειμένου;

Προς αποφυγή αργών επιδόσεων της εφαρμογής η οριζόντια μπάρα δεν προσαρμόζεται αυτόματα ανάλογα με το μήκος της γραμμής. Μπορεί να γίνει η χρήση της ιδιότητας horizontal.scroll.width ώστε να γίνει η αλλαγή της εμβέλειας της οριζόντιας κύλισης.

Πώς ενεργοποιώ το στυλ παράθυρων καρτελών στο SciTE;

Το στυλ παραθύρων καρτελών είναι διαθέσιμο μόνο σε Windows και GTK+ 2, όχι στο GTK+ 1. Το πλήθος των παραθύρων ορίζεται από ρύθμιση, για παράδειγμα, buffers=10 στο αρχείο Γενικών ρυθμίσεων (SciTEGlobal.properties). Για να φαίνεται η μπάρα των καρτελών με την εκκίνηση του SciTE, πρέπει να οριστεί το tabbar.visible=1. Μπορεί επίσης να οριστεί το tabbar.hide.one=0 για να δείχνει πάντα τις καρτέλες, ή 1 για τα τις κρύβει όταν ένα αρχείο είναι ανοχτό. Το tabbar.multiline=1 χωρίζει τις καρτέλες σε γραμμές εάν είναι απαραίτητο.

Πώς ενεργοποιώ το autocomplete;

Πηγαίνοντας στο Επιλογές | ’νοιγμα Γενικού Αρχείου Ρυθμίσεων και βγάζοντας από σχόλια την γραμμή:
autocompleteword.automatic=1

Όταν προσπαθώ να κάνω compile/build/εκτέλεση τα αρχεία κώδικα [κάποιας γλώσσας προγραμματισμού], εμφανίζεται το ακόλουθο μήνυμα λάθους: 'The system cannot find the file specified'.

Διαβεβαιώστε οτι η διαδρομή (path) του μεταγλωτιστή(compiler) είναι καθορισμένη σωστά στο σύστημα σας. Προσπαθήστε να εκτελέσετε από κονσόλα την ίδια εντολή που εκτελείτε μέσω του SciTE και δείτε αν λειτουργεί. Μπορείτε επίσης να ψάξετε στο αρχείο [γλώσσα προγραμματισμού].properties για τις εντολές μεταγλωτισμού που χρησιμοποιούνται. Αν έχετε διαφορετικό μεταγλωτιστή ή χρησιμοποιείτε διαφορετικές παραμέτρους, τροποποιήστε τις εντολές που ταιριάζουν στις ανάγκες σας. Οι σχετικές γραμμές είναι:
command.compile.filepattern=
command.build.filepattern=
command.go.filepattern=

Πώς μπορώ να προσθέσω [κάποια εξωτερική εφαρμογή] στο μενού εργαλείων του SciTE;

Στο αρχείο ιδιοτήτων σας, θα πρέπει να προσθέσετε κάποιες γραμμές:

command.name.number.filepattern
(π.χ.: command.name.1.$(file.patterns.web)=HTML Tidy)
Αυτό καθορίζει το κείμενο που θα εμφανίζεται στο μενού Εργαλεία.

command.number.filepattern
(π.χ.: command.1.$(file.patterns.web)=tidy -i -wrap 0 -m $(FilePath) )
Αυτή είναι η πραγματική εντολή που εκτελεί το SciTE. Θα πρέπει να δώσετε τις κατάλληλες διαδρομές, επιλογές και παραμέτρους όπως θα κάνατε από την γραμμή εντολών. Δείτε το SciTEDoc.html για περισσότερες πληροφορίες στις παραμέτρους και το πως να κάνετε το SciTE να εμφανίσει τον διάλογο Παράμετροι.

command.is.filter.number.filepattern
(π.χ.: command.is.filter.1.$(file.patterns.web)=1)
Η εξωτερική εφαρμογή ίσως τροποποίησε το αρχείο, επομένως αν θέσετε αυτό σε true θα κάνετε το SciTE να επαναφορτώσει το αρχείο μετά την εκτέλεση της εντολής.

command.subsystem.number.filepattern
(π.χ.: command.subsystem.1.$(file.patterns.web)=2)
Αυτό αφορά τα Windows και καθορίζει το υποσύστημα μέσω του οποίου καλείται το πρόγραμμα. Δείτε το SciTEDoc.html για περισσότερες πληροφορίες πάνω σε αυτό.

Μπορείτε να ορίσετε μια εντολή για όλα τα αρχεία χρησιμοποιώντας τον αστερίσκο (*) ως μοτίβο αρχείων. Μέχρι 10 εντολές (0 - 9) μπορούν να καθοριστούν στο μενού Εργαλεία οποιαδήποτε στιγμή. Οι εντολές αυτές μπορούν επίσης να εκτελεστούν μέσω της συντόμευσης Ctrl+αριθμός.

Υπάρχει εντολή αντικατάστασης κειμένου σε πολλαπλά αρχεία ταυτόχρονα;

Είναι πιθανό να αντικαταστήσετε κάποιο κείμενο σε όλα τα ανοιχτά παράθυρα μέσω του κουμπιού Αντικατάσταση σε Παράθυρα που βρίσκεται στον διάλογο Αντικατάσταση. Παρόλα αυτά αυτό το κουμπί είναι εξ ορισμού κρυφό, και μπορεί να εμφανιστεί με την παρακάτω γραμμή find.replace.advanced=1.

Πώς μπορώ να κάνω το SciTE να τρέχει γρηγορότερα στο GTK+;

Οι προκαθορισμένες ρυθμίσεις για το SciTE άλλαξαν στην έκδοση 1.63 ώστε να γίνει η χρήση της Pango γραμματοσειράς και για antialiased γραμματοσειρές. Μπορείτε να επιστρέψετε στην χρήση των γραμματοσειρών X core που είναι ταχύτερες με τις παρακάτω :
font.base=font:lucidatypewriter,size:12
font.small=font:lucidatypewriter,size:10
font.comment=font:new century schoolbook,size:12
font.code.comment.box=$(font.comment)
font.code.comment.line=$(font.comment)
font.code.comment.doc=$(font.comment)
font.text=font:times,size:14
font.text.comment=font:lucidatypewriter,size:10
font.embedded.base=font:lucidatypewriter,size:12
font.embedded.comment=font:lucidatypewriter,size:12
font.monospace=font:courier,size:12
font.vbs=font:new century schoolbook,size:12

Η αναδίπλωση γραμμών επίσης κάνει το SciTE να αργεί και μπορεί να απενεργοποιηθεί με wrap=0.
